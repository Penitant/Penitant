# README Redesign — Spec

**Date:** 2026-06-11  
**Status:** Approved

## Goal

Replace the current README with a clean, coding-aesthetic profile page. Remove all external GitHub API image calls. Show streak and yearly contribution count via a self-hosted SVG generated by a GitHub Action.

## Layout (top → bottom)

```
banner.svg          — pixel art, curly-haired coder at desk, night scene, 800×180
Bio block           — name + one-liner in a markdown code block
Interests           — compact inline list: AI · ML · Mathematics · CS
stats.svg           — streak count + year contribution count, updated daily by Action
LinkedIn link       — bottom, plain icon
```

## Files

| File | Purpose |
|------|---------|
| `banner.svg` | Static pixel-art banner, committed to repo |
| `stats.svg` | Generated daily by Action, committed to repo |
| `.github/workflows/update-stats.yml` | GitHub Action: cron, API query, SVG write, commit |
| `README.md` | References both SVGs as relative paths |

## banner.svg

- Dimensions: 800 × 180px
- Style: pixel art, dark night palette (deep purple/navy, city silhouette, moon)
- Scene: curly-haired figure with headphones at a desk, laptop glow, mug, warm lamp light
- No text in the SVG itself

## stats.svg

- Dimensions: 400 × 80px
- Dark card background matching banner palette
- Two metrics side by side:
  - 🔥 **Streak** — consecutive days with at least one commit
  - 📅 **{Year} Contributions** — total contributions for current calendar year
- Values written as plain text by the Action, no external font CDN

## GitHub Action

- **Schedule:** `0 2 * * *` (2 AM UTC daily)
- **Trigger:** also on `workflow_dispatch` for manual runs
- **Token:** uses built-in `GITHUB_TOKEN` (no extra secrets needed)
- **Steps:**
  1. Checkout repo
  2. Run Python script `.github/scripts/gen_stats.py`
     - Queries GitHub GraphQL API for `contributionsCollection` (year total)
     - Queries GitHub REST Events API to compute current streak
     - Writes `stats.svg` with the two values
  3. Commit and push `stats.svg` if changed (skip if no diff)

## Streak Calculation

Walk the user's `PushEvent` list from the Events API (last 90 days max, 300 events). Build a set of unique dates with at least one push. Count consecutive days back from today. If today has no push yet, allow yesterday as the streak anchor (streak isn't broken until end of day).
